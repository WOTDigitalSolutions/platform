# Client Onboarding Checklist

> Use after contract signature and initial payment.

## Pre-Kickoff
- [ ] Welcome email with summary of agreed scope and timeline.
- [ ] Share onboarding portal/folder with content upload instructions.
- [ ] Schedule vision workshop (90 minutes) and confirm participants.
- [ ] Send questionnaire (branding, messaging, media assets, integrations).

## Vision Workshop Agenda
- [ ] Ministry goals & target audience review.
- [ ] Content audit (sermons, devotionals, reports).
- [ ] Workflow mapping (attendance, follow-up, giving).
- [ ] AI feature expectations (summaries, tagging, automation).
- [ ] Success metrics validation and launch goals.

## Asset Collection
- [ ] Logos (vector + PNG), color palette, typography.
- [ ] High-resolution photos, sermon artwork, event banners.
- [ ] Existing sermon audio/video links (YouTube, SoundCloud, etc.).
- [ ] Written copy: mission, leadership bios, contact details.
- [ ] Data templates for reports, resource categories.

## Access & Integrations
- [ ] Domain registrar credentials or DNS access.
- [ ] Flutterwave/Stripe API keys (if applicable).
- [ ] Email sender/domain verification (SendGrid/Resend).
- [ ] Google Sheets or preferred reporting tool access.
- [ ] WhatsApp Business API/number for notifications (optional).

## Project Kickoff
- [ ] Share implementation timeline (Day 1–7 tasks) with milestones.
- [ ] Introduce Claude and key team members; define communication channel.
- [ ] Confirm weekly check-in schedule and escalation path.
- [ ] Provide client access to project tracker (view-only or collaborator).

## During Build
- [ ] Mid-week update (screenshots/video) for approvals.
- [ ] Track change requests; note scope impacts and timeline adjustments.
- [ ] Confirm training session date and attendee list.

## Pre-Handover
- [ ] QA checklist (responsive views, forms, payments, permissions).
- [ ] Populate initial content (sermons, resources, user accounts).
- [ ] Prepare training materials (cheat sheets, video walkthrough).
- [ ] Draft launch announcement template for client.

## Handover & Post-Launch
- [ ] Conduct live walkthrough and admin training.
- [ ] Deliver documentation (credentials, processes, support guide).
- [ ] Obtain sign-off and testimonial request (with template).
- [ ] Schedule first maintenance check-in (if applicable).
- [ ] Update CRM stage to “Portal Live” and log lessons learned.
